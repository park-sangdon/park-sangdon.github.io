AxGS Lab

How the lab is run

The lab is designed as a problem-solving unit rather than a paper-only unit.

Problem definition first

The priority is not the model name or tool stack. The first step is always to define the constraint and the metric clearly.

Code and writing together

Experimental code, production code, slides, and paper drafts should follow the same release discipline whenever possible.

AI-assisted development as a research tool

Vibe coding is used not as a gimmick, but as a way to run more experiments and shorten the loop between idea and implementation.

A stricter definition of done

Work is only considered done when operation, metrics, and reproducibility are checked. A concept without a verified artifact stays in progress.

Research Tracks

Three AxGS Lab tracks

Undergraduate researchers, capstone teams, and prospective graduate students usually anchor their work in one of these tracks.

AI x Games Systems

Building character interaction, play loops, and operational tools where AI is part of the system rather than an add-on.

Trustworthy AI

Measuring and controlling uncertainty, hallucination, and policy risks under real service conditions.

Optimization

Connecting mathematical models for pricing, scheduling, and resource allocation to actual system experiments.

Solo Development

Hexagon Soup

An ongoing solo development project that connects AI-based design and systems experiments to a real player experience.

Why a game project

Games are a strong testbed for responsiveness, cost, character design, and engagement at the same time.

How it fits the research

Planning, system implementation, balancing, and iterative play tests are handled directly so the gap between research and product stays visible.
